# 1，是啥

Caddy 是一个用 Go 编写的现代化 Web 服务器。

以其自动 HTTPS（通过 Let's Encrypt）和简洁的配置语法而闻名。

反正我用来做服务集群的入口域名的自动管理证书。

一并管一管进入的请求去向哪个端口的服务。

# 2，管理

## 1.启动

前台启动，就是会炸控制台的，方便看看啥情况。

start是后台启动，看不到控制台。

然后指定应用配置文件。

```bash
caddy_windows_amd64.exe run --config Caddyfile
```

## 2.关闭

然后用stop停止服务。

```bash
caddy_windows_amd64.exe stop
```

## 3.重加载配置

比如配置文件更新啦。

用reload应用新的配置，不用重启。

```bash
caddy_windows_amd64.exe reload
```

# 3，配置语法

## 1.基本块

全局配置块应用于所有地址的请求。

```
{
	# 全局配置块
}
```

站点配置块应用于指定域名的请求。

```
xxx.com {
	# 站点配置块
}
```

## 2.反向代理

将进入的请求干进指定端口的服务中。

```
xxx.com {
	reverse_proxy localhost:3000
}
```
